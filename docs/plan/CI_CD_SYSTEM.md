## 2. CI Pipeline

```yaml
# .github/workflows/ci.yml
stages:
  - lint:      # 代码检查
  - test:      # 单元测试
  - coverage:  # 覆盖率
  - security:  # 安全扫描
```

### 2.1 Lint Stage
- Black 格式化
- Flake8 检查
- MyPy 类型检查

### 2.2 Test Stage
- pytest 单元测试
- 覆盖率报告
- 并行执行

### 2.3 Integration Stage
- 跨模块测试
- SwarmFly FLY 层测试
- Session 集成测试

## 3. CD Pipeline

```yaml
# .github/workflows/cd.yml
deploy:
  - docker:    # Docker 镜像构建
  - pypi:      # PyPI 发布
  - release:   # GitHub Release
```

## 4. 测试策略

### 4.1 测试金字塔
```
           ┌─────┐
           │ E2E │
         ┌─┴─────┴─┐
         │集成测试 │
       ┌─┴─────────┴─┐
       │   单元测试   │
     ┌─┴─────────────┴─┐
     │    静态分析     │
     └─────────────────┘
```

### 4.2 覆盖率要求
| 层级 | 目标覆盖率 |
|------|------------|
| Core | ≥ 80% |
| SwarmFly | ≥ 70% |
| Integration | ≥ 60% |

## 5. Agent Team CI

针对多 Agent 协作场景的测试：

| 测试类型 | 场景 | 验证点 |
|----------|------|--------|
| 任务分派 | Master → Sub | 分派正确性 |
| 结果汇总 | Sub → Master | 完整性 |
| 冲突解决 | 资源竞争 | 仲裁正确性 |
| 异常恢复 | 节点失败 | 自愈能力 |
