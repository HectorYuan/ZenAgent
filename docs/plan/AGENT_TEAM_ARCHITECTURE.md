# Agent Team 架构设计

> 整合自：Knowledge/AgentCI_AgentTeam架构设计方案_v2.1.md, Knowledge/AgentCI_面向AgentTeam轻量化方案_v2.0.md

## 1. 架构概述

Agent Team 是一个多智能体协作系统，支持主从架构、层级结构和网状协作。

```
┌─────────────────────────────────────────────────────┐
│                    Master Agent                      │
│  (任务理解 / 分派 / 验收)                            │
└────────────────┬────────────────────────────────────┘
                 │
    ┌────────────┼────────────┐
    ▼            ▼            ▼
┌────────┐  ┌────────┐  ┌────────┐
│ Team A │  │ Team B │  │ Team C │
└────────┘  └────────┘  └────────┘
```

## 2. 核心组件

### 2.1 Master Agent (主智能体)
- 任务理解与分解
- 子智能体分派
- 结果验收与汇总

### 2.2 Team Leader (团队领导)
- 团队内任务协调
- 资源分配
- 进度跟踪

### 2.3 Sub Agent (子智能体)
- 独立任务执行
- 状态上报
- 结果返回

## 3. 协作模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| 主从模式 | 串行任务 | Master 统一调度 |
| 层级模式 | 复杂任务分解 | 多层 Team Leader |
| 网状模式 | 并行独立任务 | Agent 间直接通信 |

## 4. 通信机制

### 4.1 消息格式
```json
{
  "agent_id": "agent_001",
  "task_id": "task_001",
  "message_type": "task/submit/complete/error",
  "payload": {},
  "timestamp": "2026-05-14T08:00:00Z"
}
```

### 4.2 会话管理
- 每个 Agent 维护独立 Session
- Session 间共享 Memory
- 支持跨 Session 任务延续

## 5. SwarmFly 集成

SwarmFly 提供底层架构支撑：

| FLY 层 | 功能 | Agent Team 支撑 |
|--------|------|----------------|
| L1 使命层 | 目标分解 | 任务理解与拆分 |
| L2 法则层 | 规则引擎 | 协作协议执行 |
| L3 趋势层 | 自适应 | 动态资源调度 |
| L4 技能层 | 能力注册 | Agent 技能管理 |
| L5 工具层 | 资源池 | 消息队列/工具调用 |
