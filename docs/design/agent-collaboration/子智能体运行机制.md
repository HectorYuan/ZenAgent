## 一、核心公式与架构

### 1.1 智能体定义公式

```
Agent = Model + Harness + Skill + Calendar

├── Model：大模型核心（提供基础智能）
├── Harness：驾驭层（运行时控制系统）
│   ├── 工具集成层：统一管理所有工具调用
│   ├── 上下文工程：动态管理对话上下文
│   ├── 状态持久化：任务断点续传、状态恢复
│   ├── 安全防护：权限控制、异常兜底
│   └── 可观测性：执行追踪、审计日志
├── Skill：技能层（专业技能承载）
└── Calendar：日程层（任务调度驱动）
```

### 1.2 五层架构定位

```
L5 修炼目标层（Why）── 境界突破 · 知识沉淀 · 技能提升
        ↓
L4 智能体集群层（Who）── 主智能体 + 子智能体集群 ⭐
        ↓
L3 功能框架层（How）── HyperHarness 引擎
        ↓
L2 驱动服务层（Driver）── D1-D4 四层驱动
        ↓
L1 基础设施层（Foundation）── Coze平台 · 工具集 · 技能库
```

### 1.3 架构层次说明

| 层次 | 名称 | 核心职责 | 更新成本 | 更新频率 |
|------|------|----------|----------|----------|
| **L5** | 修炼目标层 | 境界突破、知识沉淀、技能提升、价值实现 | 极低 | 周度级 |
| **L4** | 智能体集群层 | 主智能体+子智能体集群、任务分派与执行 | 低 | 月度级 |
| **L3** | 功能框架层 | HyperHarness引擎、B1-B13模块、C1-C8机制 | 中等 | 季度级 |
| **L2** | 驱动服务层 | D1-D4四层驱动、脚本体系、安全控制 | 高 | 季度级 |
| **L1** | 基础设施层 | Coze平台、工具集、技能库、知识库 | 极高 | 几乎不变 |

### 2.2 创造机制（动态运行）

**创建工具**：`sessions_spawn`

**创建流程**：

```
┌─────────────────────────────────────────────────────────────┐
│                    子智能体创建流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 触发条件                                                 │
│     ├── 用户指令：用户明确要求某任务                           │
│     ├── 关键词路由：根据关键词自动路由                         │
│     ├── 日程触发：日程时间到达                                │
│     └── 协作链：上一节点完成触发                              │
│                                                             │
│  2. 主智能体识别任务                                          │
│     ├── 理解任务需求                                         │
│     ├── 确定目标智能体                                       │
│     └── 编写任务描述                                         │
│                                                             │
│  3. 调用 sessions_spawn                                      │
│     ├── name: 智能体名称（简短）                              │
│     ├── task: 任务描述（完整上下文）                          │
│     ├── agent: 使用哪个agent（默认lead）                      │
│     └── run_timeout_seconds: 超时时间                        │
│                                                             │
│  4. 子智能体启动                                              │
│     ├── 共享主智能体全部能力                                  │
│     ├── 共享文件系统                                         │
│     ├── 共享记忆文件                                         │
│     └── 独立session执行                                      │
│                                                             │
│  5. 执行与反馈                                                │
│     ├── 子智能体独立执行任务                                  │
│     ├── 产出文件到指定目录                                    │
│     ├── 执行完成后异步推送结果                                │
│     └── 主智能体验收与通知用户                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**创建示例**：

```python
# 主智能体调用
sessions_spawn(
    name="产品需求分析",
    task="""
    项目背景：用户需要新增用户登录功能
    任务目标：撰写PRD文档
    
    执行步骤：
    1. 分析用户登录需求
    2. 撰写用户故事
    3. 输出PRD文档
    
    产出目录：Resources/产品方案/
    产出文件：PRD-用户登录-v1.0.md
    
    使用技能：design-assistant, docx
    """,
    run_timeout_seconds=300
)
```

### 2.3 触发条件详解

| 触发方式 | 说明 | 示例 |
|----------|------|------|
| **用户指令** | 用户明确要求某协作链 | "帮我设计一个新功能" |
| **关键词路由** | 根据关键词自动路由到起始智能体 | "产品需求"、"架构设计" |
| **文档传递** | 通过文档交接触发下一节点 | PRD完成后触发架构设计 |
| **日程触发** | 定时触发协作链执行 | 每日投资分析 |
| **协作链流转** | 上一节点完成后触发 | 架构设计完成→开发者 |

## 三、生命周期管理

### 3.1 生命周期状态

```
┌──────────┐    sessions_spawn    ┌──────────┐
│  未创建  │ ─────────────────→  │  运行中   │
└──────────┘                      └────┬─────┘
                                       │
                    ┌──────────────────┼──────────────────┐
                    │                  │                  │
                    ▼                  ▼                  ▼
              ┌──────────┐      ┌──────────┐      ┌──────────┐
              │  已完成   │      │  阻塞中   │      │  已中止   │
              └──────────┘      └──────────┘      └──────────┘
                    │                  │
                    │                  │ 问题解决
                    │                  ▼
                    │            ┌──────────┐
                    │            │  已恢复   │
                    │            └──────────┘
                    │
                    ▼
              结果推送给主智能体
```

### 3.2 状态说明

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| **未创建** | 子智能体尚未启动 | 初始状态 |
| **运行中** | 子智能体正在执行任务 | sessions_spawn 调用 |
| **已完成** | 任务成功完成 | 子智能体执行完毕 |
| **阻塞中** | 任务遇到问题无法继续 | 资源不足、依赖缺失 |
| **已恢复** | 问题解决后恢复执行 | 阻塞问题解决 |
| **已中止** | 任务被主动终止 | sessions_abort 调用 |

### 3.3 生命周期管理工具

| 工具 | 功能 | 使用场景 |
|------|------|----------|
| `sessions_spawn` | 创建子智能体 | 启动新的sub-agent session |
| `sessions_send` | 发送消息 | 向子智能体发送指令、传递上下文 |
| `sessions_history` | 获取历史 | 查看子智能体执行记录 |
| `sessions_list` | 列表查询 | 查看所有运行中的session |
| `sessions_status` | 状态查询 | 查看特定session状态 |
| `sessions_abort` | 中止任务 | 终止正在运行的session |

### 3.4 会话信息管理

```
会话类型：
├── main：主对话（与用户直接交互）
├── cron：日程触发会话
└── sub：子智能体会话

会话状态：
├── idle：空闲
├── running：运行中
└── waiting：等待中
```

## 六、权限与协作机制

### 6.1 权限机制 ⚠️ 重要

**核心发现**：sub-agent 的工具权限是**系统级限制**，而非配置问题。

```
┌─────────────────────────────────────────────────────────────┐
│                    权限管理机制                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【权限来源】                                                 │
│  sessions_spawn 创建的所有 sub-agent 共享同一套系统预设工具集  │
│                                                             │
│  【权限限制】                                                 │
│  - 无法动态分配不同工具权限                                   │
│  - 权限必须在创建时由主智能体授予                             │
│  - 需要设计权限的动态生命周期管理                             │
│                                                             │
│  【解决方案】                                                 │
│  1. 任务总线注册表机制                                       │
│     └── TASK-SYSTEM.json 注册所有任务                         │
│  2. 技能路由器协调                                           │
│     └── 匹配智能体与技能                                     │
│  3. 提权到主智能体执行                                       │
│     └── 子智能体无法执行时由主智能体接管                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 协作机制

#### 6.2.1 任务总线机制

```
任务总线（Task Bus）：
├── 配置文件：System/.shared_state/TASK-SYSTEM.json

└── 包含：任务队列、状态跟踪、智能体分配

#### 6.2.2 协作链机制

详细内容参见：`Resources/知识库/L5-Templates/智能体协作/协作链指南.md`

#### 6.2.3 消息传递机制

```
消息类型：
├── 任务指令：主智能体 → 子智能体
├── 执行状态：子智能体 → 主智能体
