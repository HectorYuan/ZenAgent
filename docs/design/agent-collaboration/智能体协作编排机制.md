## 一、机制概述

### 1.1 设计背景

随着赛博游侠智能体集群的不断扩展，方案评审流程日趋复杂。为了实现：
- **职责分离**：决策层与执行层解耦
- **配置驱动**：角色、权重、阈值可动态调整
- **可观测性**：完整的执行追踪和结果记录

设计了这套完整的智能体协作编排机制。

### 1.2 核心设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                      决策层分离架构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌─────────────┐      ┌──────────────────┐      ┌────────────┐ │
│   │  协作引擎   │ ──── │     主 Agent      │ ──── │  子智能体  │ │
│   │  (计算层)   │      │    (执行层)       │      │  (执行器)  │ │
│   └─────────────┘      └──────────────────┘      └────────────┘ │
│         │                       │                        │      │
│         ▼                       ▼                        ▼      │
│   - 评审计划生成              - sessions_spawn          - 任务   │
│   - 权重计算                  - 结果收集                - 评分   │
│   - 结果聚合                  - 流程控制                - 输出   │
│   - 决策计算                  - 异常处理                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 核心特点

| 特点 | 说明 |
|------|------|
| **决策层分离** | 协作引擎只负责计算和规划，不执行实际调用 |
| **主 Agent 执行** | sessions_spawn 调用由主 Agent 直接执行 |
| **配置驱动** | 角色、权重、阈值通过配置文件管理 |
| **可观测性** | 完整的执行追踪、结果记录、风险评估 |
| **灵活性** | 支持顺序执行、并行执行、快速执行三种模式 |

### 4.2 评审计划结构

```python
@dataclass
class ReviewPlan:
    collaboration_id: str      # 协作会话ID
    proposal_id: str           # 方案ID
    proposal_type: str         # 方案类型
    execution_mode: ExecutionMode  # 执行模式
    roles: List[RoleWeight]    # 角色配置列表
    weights: Dict[str, float]  # 权重配置
    threshold: float          # 通过阈值
```

### 4.3 角色任务结构

```python
@dataclass
class RoleTask:
    role_id: str               # 角色ID
    role_name: str             # 角色名称
    agent: str                 # 智能体名称
    task_description: str      # 任务描述
    timeout_seconds: int       # 超时时间
    required: bool             # 是否必需
    previous_outputs: List[str] # 前置角色输出
```

### 4.4 聚合算法

```
聚合得分 = Σ(角色评分 × 角色权重) / Σ(角色权重)

异常值处理：
1. Z-score 检测：|z| > 2.0 判定为异常
2. 处理方式：trim（裁剪到正常范围）
```

### 4.5 决策逻辑

```
决策树：
├── 评分 >= 0.75 → 通过 (APPROVE)
├── 评分 >= 0.60 → 有条件通过 (CONDITIONAL_APPROVE)
├── 评分 >= 0.40 → 需修改 (NEEDS_REVISION)
└── 评分 < 0.40 → 不通过 (REJECT)

人工介入条件：
├── 综合得分 < 0.50
├── 置信度 < 50%
├── 评分分歧过大 (variance > 0.1)
└── 存在异常值
```

## 五、执行层设计

### 5.1 主 Agent 调用流程

```python
# 主 Agent 中的伪代码
def execute_collaboration_review(proposal):
    """执行协作评审"""
    
    # ========== 计算层 ==========
    # 1. 初始化协作引擎
    engine = CollaborationEngine()
    
    # 2. 生成评审计划
    plan = engine.get_review_plan(proposal)
    
    # 3. 生成执行计划
    execution_plan = engine._generate_execution_plan(plan)
    
    # 4. 获取任务模板
    templates = engine._get_task_templates_for_plan(plan)
    
    # ========== 执行层 ==========
    # 5. 按计划执行角色调用
    results = {}
    for step in execution_plan["steps"]:
        role = step["role_id"]
        
        # 构建任务
        task = engine.build_task_for_role(role, proposal)
        
        # 执行调用（由主 Agent 执行）
        session = sessions_spawn(
            name=f"{task.role_name}评审",
            task=task.task_description,
            agent=task.agent,
            run_timeout_seconds=task.timeout_seconds
        )
        
        # 收集结果
        result = wait_for_result(session)
        results[role] = result
    
    # ========== 计算层 ==========
    # 6. 聚合结果
    aggregated = engine.aggregate_results(results, plan=plan)
    
    # 7. 做出决策
    decision = engine.make_decision(aggregated, plan=plan)
    
    # 8. 生成报告
    report = engine.process_results(results, plan)
    
    return report
```

### 5.2 执行模式

#### 顺序执行 (Sequential)

```
角色1 ──> 角色2 ──> 角色3 ──> 角色4 ──> 角色5
         ↑                    ↑
         └────── 上下文传递 ──────┘

适用场景：架构级方案
- 角色间有依赖关系
- 需要前序角色结果作为后续输入
- 评审质量要求高
```

#### 并行执行 (Parallel)

```
┌─────────┐  ┌─────────┐  ┌─────────┐
│ 角色1   │  │ 角色2   │  │ 角色3   │
└─────────┘  └─────────┘  └─────────┘
                 │
                 ▼
           ┌─────────┐
           │ 角色4   │
           └─────────┘
                 │
                 ▼
           ┌─────────┐
           │ 角色5   │
           └─────────┘

适用场景：工具级方案
- 无依赖关系的角色同时执行
- 提高执行效率
- 适用于简单方案
```

#### 快速执行 (Quick)

```
角色1 ──> 角色2 ──> 决策

适用场景：流程级方案
- 仅核心角色参与
- 最短时间完成评审
- 适用于紧急或简单场景
```

### 5.3 执行状态跟踪

```
状态流转：
pending ──> in_progress ──> completed
                │
                ├──> failed ──> in_progress (重试)
                │
                └──> degraded ──> completed
```

### 6.2 评分计算

```
综合得分 = 加权平均(角色评分, 角色权重)

其中：
- 权重归一化：Σ(权重) = 1.0
- 异常值处理：trim 到 [mean-1.5*stdev, mean+1.5*stdev]
- 置信度：基于角色数量、异常值、评分方差计算
```

### 6.3 决策类型

| 决策 | 得分范围 | 说明 | 后续动作 |
|------|----------|------|----------|
| **通过** | 0.75-1.0 | 方案可直接执行 | 进入实施阶段 |
| **有条件通过** | 0.60-0.75 | 方案需修改后执行 | 提交修改后重新评审 |
| **需修改** | 0.40-0.60 | 方案需要重大修改 | 返回方案设计阶段 |
| **不通过** | 0.0-0.40 | 方案不适合执行 | 归档或终止 |

## 九、使用示例

### 9.1 主 Agent 调用示例

```python
# 主 Agent 中执行协作评审
def handle_proposal_review(user_request):
    """处理用户的方案评审请求"""
    
    # 1. 解析方案信息
    proposal = {
        "proposal_id": "PROP-001",
        "proposal_name": "任务调度优化",
        "proposal_type": "architecture",
        "content": user_request
    }
    
    # 2. 初始化协作引擎
    engine = CollaborationEngine()
    
    # 3. 生成评审计划
    plan = engine.get_review_plan(proposal)
    
    # 4. 执行评审
    results = {}
    for step in plan.execution_steps:
        task = engine.build_task_for_role(step.role_id, proposal)
        
        # 执行真实的 sessions_spawn
        session = sessions_spawn(
            name=f"{task.role_name}评审",
            task=task.task_description,
            agent=task.agent,
            run_timeout_seconds=task.timeout_seconds
        )
        
        # 收集结果
        results[step.role_id] = wait_for_result(session)
    
    # 5. 聚合和决策
    aggregated = engine.aggregate_results(results, plan=plan)
    decision = engine.make_decision(aggregated, plan=plan)
    
    # 6. 返回结果
    return {
        "success": True,
        "decision": decision,
        "results": results
    }
```

# 检查必需角色
for role_id, result in results.items():

## 十四、主对话方案自动化集成

### 14.1 设计目标

在主对话中实现完整的方案管理自动化闭环：

```
用户提出方案 → 自动检测 → 自动注册 → 自动评审 → 结果反馈
```

### 14.2 核心组件

| 组件 | 文件路径 | 职责 |
|------|----------|------|
| **方案检测器** | `System/.shared_state/scripts/proposal_flow.py` | 检测用户消息中的方案内容 |
| **方案注册器** | `System/.shared_state/scripts/proposal_flow.py` | 注册方案并分配唯一ID |
| **流程控制器** | `System/.shared_state/scripts/proposal_flow.py` | 协调全流程执行 |
| **状态文件** | `System/.shared_state/proposal_flow_state.json` | 跟踪流程状态 |
| **方案注册** | `System/.shared_state/scripts/proposals_registry.json` | 存储已注册方案 |

### 14.3 触发规则

#### 触发条件

| 条件类型 | 要求 |
|----------|------|
| **消息长度** | ≥200字符（普通） 或 ≥50字符（有明确信号） |
| **关键词** | 方案、设计方案、优化方案、实施方案 等 |
| **明确信号** | "我有一个方案"、"设计一个方案" 等 |

#### 方案类型自动识别

| 类型 | 关键词 | 执行模式 |
|------|--------|----------|
| **架构级** | 架构、系统架构、整体设计 | 顺序执行 |
| **模块级** | 模块、功能模块、子系统 | 顺序执行 |
| **工具级** | 工具、脚本、小工具 | 并行执行 |
| **流程级** | 流程、工作流、自动化流程 | 快速执行 |

### 14.4 主 Agent 使用接口

#### 方式A：便捷函数（推荐）

```python
# 导入便捷函数
from proposal_flow import (
    handle_proposal_in_conversation,
    execute_review_plan,
    finalize_review,
    get_current_state,
    reset_flow_state
)

# Step 1: 检测并注册方案
result = handle_proposal_in_conversation(user_message)

if result["detected"]:
    proposal_id = result["proposal_id"]
    plan = result["plan"]
    
    # Step 2: 获取评审角色列表
    roles = execute_review_plan(plan)
    
    # Step 3: 主 Agent 执行 sessions_spawn
    results = {}
    for role in roles:
        session = sessions_spawn(
            name=f"{role['role_name']}评审",
            task=build_task_for_role(role, plan),
            agent=role["agent"],
            run_timeout_seconds=role["timeout_seconds"]
        )
        results[role["role_id"]] = wait_for_result(session)
    
    # Step 4: 聚合结果并完成流程
    final = finalize_review(proposal_id, results)
    
    # Step 5: 反馈给用户
    inform_user(final["decision"]["decision_type"], final["aggregated"]["total_score"])
```
