## 二、核心角色定义

### 2.1 主智能体 (Master Agent)

**定位**：协作系统的中枢大脑，负责全局协调和决策

| 组件 | 功能 | 职责 |
|------|------|------|
| **协商器** | 任务分解 + 角色分配 | 将复杂任务拆分为子任务，分配给合适的子智能体 |
| **路由器** | 智能路由 + 负载均衡 | 根据智能体能力和负载选择最优执行路径 |
| **仲裁器** | 冲突检测 + 解决方案 | 检测子智能体间的冲突，进行裁决 |
| **共识器** | 投票机制 + 决策聚合 | 收集多方意见，形成统一决策 |

### 2.2 主智能体工作流程

```
用户请求
    ↓
任务分析 ─────────────────────────────────────────┐
    ↓                                                │
任务拆分 ───→ 子任务1, 子任务2, 子任务3            │
    ↓                                                │
能力匹配 ───→ Registry查询最优智能体                │
    ↓                                                │
协商分配 ───→ 分发任务给子智能体群                  │
    ↓                                                │
并行执行 ───→ 子智能体独立/协作执行                │
    ↓                                                │
结果聚合 ───→ 汇总各子智能体结果                    │
    ↓                                                │
质量验证 ───→ ZenLoop太极平衡校准                   │
    ↓                                                │
最终输出 ◀────────────────────────────────────────┘
```

## 三、子智能体群 (Agent Swarm)

### 3.1 定位

执行层面的多个专业化智能体集群

### 3.2 智能体类型分类

| 类型 | 标识色 | 示例 | 能力 |
|------|--------|------|------|
| **Research** | 🟢 绿色 | 研究分析Agent | 信息检索、数据分析、报告生成 |
| **Writer** | 🟡 黄色 | 内容创作Agent | 文案撰写、文档编辑、翻译 |
| **Code** | 🔴 红色 | 代码开发Agent | 代码生成、调试、重构 |
| **Data** | 🟢 绿色 | 数据处理Agent | ETL、可视化、统计 |
| **Design** | 🟡 黄色 | 设计Agent | UI设计、原型制作 |
| **PM** | 🔴 红色 | 项目管理Agent | 任务跟踪、进度协调 |

### 3.3 子智能体群特点

- **并行执行**：多个子智能体可同时工作
- **能力互补**：不同类型智能体擅长不同领域
- **动态协作**：可根据任务需要组成临时团队
- **状态感知**：实时上报负载和健康状态

## 四、协作模式

### 4.1 主从协作（星型拓扑）

```
        👑 Master
       /    |    \
      /     |     \
     ↓      ↓      ↓
  [Agent] [Agent] [Agent]
  
适用场景：简单任务、单一领域、快速执行
```

### 4.2 协商协作（网状拓扑）

```
        👑 Master
       /    |    \
      ↓     ↓      ↓
   [A1]←→[A2]←→[A3]
      ↑    ↓    ↑
      └──[A4]───┘
   
适用场景：复杂任务、需要多方协调
```

### 4.3 流水线协作（链式拓扑）

```
[A1] → [A2] → [A3] → [A4]
 ↑       ↑       ↑       ↑
分析   写作   审核   输出

适用场景：顺序依赖任务、多阶段处理
```

### 4.4 涌现协作（动态拓扑）

```
      👑 Master (监控)
         ↓ 观察
    ┌────┼────┐
    ↓    ↓    ↓
[A1]  [A2]  [A3]
    ↘  ↓  ↙
    [A4] ←→ [A5]
        ↑
   (自主涌现协作)
   
适用场景：复杂问题、开放性任务
```
