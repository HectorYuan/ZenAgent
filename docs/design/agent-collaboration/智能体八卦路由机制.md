## 执行摘要

本文档定义了ZenLoop智能体系统的**八卦路由机制**，是系统消息路由的核心规范。

### 核心概念
- **路由策略**：双轨协同——功能协作轨（60%）+ 五行能量轨（40%）
- **八卦相生**：乾/兑→坎→震/巽→离→艮/坤→乾/兑（职责驱动）
- **八卦相克**：乾/兑→震/巽→艮/坤→坎→离→乾/兑（需规避）

### 关键参数
| 参数 | 值 |
|------|-----|
| 最短路径权重 | 40% |
| 负载均衡权重 | 30% |
| 八卦匹配权重 | 20% |
| 相位最优权重 | 10% |
| 最大跳数限制 | 3跳 |
| 八卦匹配阈值 | >0.7 |

### 决策层级
- **战略层**（双轨裁决）：任务分配策略与能量平衡优先级
- **战术层**（路由决策）：具体消息的最优传输路径

### 文档关联
- 上游：太极两仪涌现机制（ZenLoop核心）
- 下游：智能体集群运行机制（消息协议执行）

### 2.1 【路由决策冲突裁决规则】

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      路由决策冲突裁决规则                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【冲突场景】                                                               │
│  当 S_协作 和 S_能量 计算结果指向不同目标时，发生路由决策冲突               │
│                                                                             │
│  【冲突裁决规则】                                                           │
│  ┌─────────────────────────────────────────────────────────────────────┐    │

│  │                                                                      │    │
│  │  规则2：任务优先级 >= P2 → 功能协作轨优先                           │    │
│  │         S_total = S_协作（忽略S_能量）                              │    │
│  │                                                                      │    │
│  │  规则3：其他情况 → 战略层加权求和                                   │    │

│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  【冲突检测算法】                                                           │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  1. 计算 |S_协作 - S_能量| 差值                                     │    │
│  │                                                                      │    │
│  │  2. 若差值 > 0.3：                                                  │    │
│  │     → 差值大的轨道获胜（强主导）                                    │    │
│  │                                                                      │    │
│  │  3. 若差值 <= 0.3：                                                 │    │
│  │     → 战略层加权求和决定（均衡决策）                               │    │
│  │                                                                      │    │
│  │  【差值阈值说明】                                                   │    │
│  │  ├── 差值 > 0.3：明确冲突，强轨主导                               │    │
│  │  └── 差值 <= 0.3：轻微分歧，协商决策                               │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                             │
│  【冲突解决示例】                                                           │
│  ├── 场景：S_协作=0.8(目标A), S_能量=0.3(目标B)，差值=0.5 > 0.3          │
│  │       → 目标A获胜（协作轨主导）                                     │
│  │                                                                      │    │
│  └── 场景：S_协作=0.6(目标A), S_能量=0.5(目标B)，差值=0.1 <= 0.3         │
```

### 2.4 路由决策公式

```
综合路由分数 = W1×S_dijkstra + W2×S_load + W3×S_match + W4×S_phase + W5×S_energy

其中：
├── S_dijkstra（最短路径分）= 1 - (实际跳数 / 最大跳数)
├── S_load（负载均衡分）= 1 - (当前负载 / 容量上限)
├── S_match（八卦匹配分）= CosineSimilarity(消息向量, 智能体向量)
├── S_phase（相位最优分）= 1 - (相位差 / 180°)
└── S_energy（五行能量分）= 能量平衡因子 = 1 - |ζ_source - ζ_target| / 100

权重配置：W1=0.3, W2=0.2, W3=0.2, W4=0.1, W5=0.2
```
