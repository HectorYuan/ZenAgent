
### 1.2 核心公式

```
Agent = Model + Harness + Skill + Calendar
```

**智能体集群采用主智能体 + 子智能体集群的协作模式**：
- **主智能体**（Lead Agent）：任务理解、分派、验收、沟通协调
- **子智能体集群**（12个）：专注领域执行、技能承载、价值产出

### 1.3 三层协作模型

```
┌─────────────────────────────────────────────────────┐

         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
┌─────────────┐   ┌─────────────┐   ┌─────────────┐
│ 计划执行组   │   │ 任务执行组   │   │ 系统支持组   │
```

### 3.4 任务优先级与六车道映射

**六车道完整定义**：

| 车道 | 方向 | 优先级 | 生命周期 | 晋升策略 | 调度频率 | 并发数 | 典型任务 |
|------|------|--------|---------|---------|---------|-------|----------|
| 🚀 FAST-P | 阳 | P0/P1 | 即时 | 自动晋升+立即触发 | 实时 | 1 | 紧急故障修复、用户阻塞问题 |
| 🚗 NORMAL-P | 阳 | P2/P3 | 中期(1-7天) | 自动晋升 | 每日06:00 | 1 | 功能开发、迭代任务 |
| 🚶 SLOW-P | 阳 | P4/P5 | 长期(周/月) | 手动触发 | 每周日22:00 | 1 | 技术规划、架构演进 |
| 🚀 FAST-E | 阴 | CRITICAL | 分钟级 | 阻塞执行 | 实时 | 1 | 紧急执行、关键路径 |
| 🚗 NORMAL-E | 阴 | HIGH/NORMAL | 小时级 | 标准调度 | 每5分钟 | 3 | 常规执行、批量处理 |
| 🚶 SLOW-E | 阴 | LOW | 天级 | 空闲调度 | 每日02:00 | 1 | 低优先级、后台任务 |

**车道选择指南**：

```
用户请求/日程触发
       ↓
┌─────────────────────────────────────────┐
│ 紧急程度判断                              │
├─────────────────────────────────────────┤
│ 🚀 P0/P1 → 🚀 FAST-P → 🚀 FAST-E       │
│   立即执行，阻塞式处理                     │
├─────────────────────────────────────────┤
│ 🚗 P2/P3 → 🚗 NORMAL-P → 🚗 NORMAL-E   │
│   标准调度，排队执行                       │
├─────────────────────────────────────────┤
│ 🚶 P4/P5 → 🚶 SLOW-P → 🚶 SLOW-E        │
│   低优先级，空闲执行                       │
└─────────────────────────────────────────┘
```

**响应时间要求**：

| 车道 | 响应时间 | 说明 |
|------|----------|------|
| 🚀 FAST-P/E | <5分钟 | 紧急任务，即时响应 |
| 🚗 NORMAL-P/E | <30分钟 | 标准任务，尽快处理 |
| 🚶 SLOW-P/E | <24小时 | 低优先级，空闲处理 |

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           双向六车道体系                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   【execution】规划方向 ──────────────────────────────────────────────────────→    │
│   ┌─────────────────┬─────────────────┬─────────────────┐                   │
│   │    🚀 FAST-P    │   🚗 NORMAL-P   │    🚶 SLOW-P    │                   │
│   │    P0/P1        │     P2/P3       │     P4/P5       │                   │
│   │    即时创建      │    每日晋升      │     手动晋升    │                   │
│   │    立即晋升      │    每日06:00     │     周检视      │                   │
│   └─────────────────┴─────────────────┴─────────────────┘                   │
│                                                                              │
│   ════════════════════════════════════════════════════════════════          │
│                         【太极】阴阳分界                                         │
│   ════════════════════════════════════════════════════════════════          │
│                                                                              │
│   ←─────────────────────────────────────────────────────── 【sync】执行方向  │
│   ┌─────────────────┬─────────────────┬─────────────────┐                   │
│   │    🚀 FAST-E    │   🚗 NORMAL-E   │    🚶 SLOW-E    │                   │
│   │    CRITICAL     │   HIGH/NORMAL   │      LOW        │                   │
│   │    阻塞式执行    │    标准调度      │    空闲调度      │                   │
│   │    并发数=1      │    并发数=3      │    并发数=1      │                   │
│   └─────────────────┴─────────────────┴─────────────────┘                   │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 各车道状态流转

#### 🚀 FAST-P 状态流转
```
CREATED → READY → PROMOTED → EXECUTING → COMPLETED
             ↓           ↓           ↓
         BLOCKED     RUNNING      FAILED
             ↓           ↓
             └───────────┴──→ SLOW-P (降级)
```

#### 🚗 NORMAL-P 状态流转
```
CREATED → READY → WAITING → READY → PROMOTED → QUEUED → EXECUTING → COMPLETED
                      ↓
                  DEFERRED
```

#### 🚶 SLOW-P 状态流转
```
CREATED → READY → WAITING → HOLD → ARCHIVED
                   ↓
               PROMOTED (手动触发)
```

#### 🚀 FAST-E 状态流转（阻塞式）
```
PROMOTED → BLOCKING_WAIT → EXECUTING → COMPLETED
              ↓                    ↓
           SLOW-E            FAILED (3次重试)
```

#### 🚗 NORMAL-E 状态流转（标准）
```
PROMOTED → QUEUED → EXECUTING → COMPLETED
                           ↓
                       FAILED (2次重试)
                           ↓
                       WAITING (资源等待)
                           ↓
                       DEFERRED → SLOW-E
```

#### 🚶 SLOW-E 状态流转（空闲）
```
PROMOTED → QUEUED → EXECUTING → COMPLETED
                           ↓
                       FAILED → SKIPPED → ARCHIVED
```

### 4.3 晋升规则

| 车道 | 晋升条件 | 晋升时间 | 手动干预 |
|------|---------|---------|---------|
| 🚀 FAST-P → 🚀 FAST-E | P0/P1创建 | 实时 | 无 |
| 🚗 NORMAL-P → 🚗 NORMAL-E | P2/P3队列 | 每日06:00 | 无 |
| 🚶 SLOW-P → 🚶 SLOW-E | P4/P5手动确认 | 每周日22:00 | 需确认 |
| 任意 → 降级 | 执行失败N次 | 失败时 | 自动 |

### 4.4 并发控制

| 车道 | 最大并发 | 说明 |
|------|---------|------|
| 🚀 FAST-E | 1 | 阻塞式执行，防止资源竞争 |
| 🚗 NORMAL-E | 3 | 标准并发，平衡效率与稳定 |
| 🚶 SLOW-E | 1 | 空闲执行，避免影响其他任务 |
| 其他车道 | 1 | 均为单任务排队 |

### 5.1 单智能体任务

**场景**：生成每日市场简报

```
日程触发 → 投资分析师读取任务描述 → 
加载数据源 → 生成报告 → 保存文件 → 
通知主智能体 → 主智能体验收 → 推送飞书
```

**示例流程**:
```python
# 1. 日程触发
calendar_service.parse_agent_task_from_calendar(event)
```

## 九、故障处理

### 8.1 常见问题

#### 问题1: 任务调度失败

**现象**: 
```
status: "failed"
error: "未知智能体: xxx"
```

**原因**: 智能体ID不存在或未注册

**解决方案**:
```bash
# 1. 检查智能体注册
cat System/.shared_state/agent_registry.json | jq '.agents | keys'

# 2. 注册缺失的智能体
python3 System/.shared_state/scripts/agent_management/agent_registry.py \
  --register --agent-id "new_agent" --name "新智能体"

# 3. 更新路由矩阵
python3 System/.shared_state/scripts/agent_management/agent_router.py --update-matrix
```

#### 问题2: 技能加载失败

**现象**:
```
skill_load_success_rate: 0.0
failed_skills: ["topic_tracking", "excel_master"]
```

**原因**: 技能文件缺失或路径错误

**解决方案**:
```bash
# 1. 检查技能文件
ls -la Skills/

# 2. 检查技能状态
cat System/.shared_state/skill_loading_status.json | jq '.failed'

# 3. 重新加载技能
python3 System/.shared_state/scripts/agent_management/skill_loader.py \
  --reload --skill "topic_tracking"
```

#### 问题3: 队列积压

**现象**:
```
queue_length: 200+
pending_tasks: 150+
```

**原因**: 任务处理速度慢或服务异常

**解决方案**:
```bash
# 1. 检查服务状态
ps aux | grep "task_service"

# 2. 重启服务
systemctl restart task_service

# 3. 清理积压任务
python3 System/.shared_state/scripts/agent_management/agent_scheduler.py \
  --cleanup --older-than 7d

# 4. 增加并发
修改 dispatch_config.json: max_per_heartbeat: 10
```

#### 问题4: 智能体失联

**现象**:
```
agent_status: "offline"
last_active: "2小时前"
```

**原因**: 子智能体session异常终止

**解决方案**:
```bash
# 1. 检查session列表
python3 -c "from tools import sessions_list; print(sessions_list())"

# 2. 重启智能体
sessions_spawn(task="重新初始化", agent="investor")

# 3. 更新注册表状态
python3 System/.shared_state/scripts/agent_management/agent_registry.py \
  --update-status --agent-id "investor" --status "idle"
```

### 8.2 异常处理流程

#### 任务执行失败

**处理流程**:
```
子智能体报告失败 → 主智能体分析原因 →
判断是否可重试 →
  ├─ 可重试：调整参数后重新派发（最多3次）
  └─ 不可重试：降级处理，告知用户
```

#### 资源不可用

**处理策略**:
| 资源类型 | 不可用时处理 |
|----------|--------------|
| 文件 | 创建替代文件或跳过 |
| 技能 | 使用替代方案或降级 |
| 外部API | 重试或使用缓存数据 |
| 智能体 | 路由到备用智能体 |

#### 上下文丢失

**恢复机制**:
1. 从MEMORY.md恢复对话记忆
2. 从文件系统恢复状态
3. 从日志恢复执行进度
4. 重新注入上下文信息
