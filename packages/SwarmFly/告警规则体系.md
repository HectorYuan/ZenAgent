## 一、告警规则概述

### 1.1 告警等级

| 等级 | 标识 | 颜色 | 说明 | 响应时间 |
|------|------|------|------|----------|
| P0 | CRITICAL | 红色 | 致命，系统不可用 | 5分钟内 |
| P1 | ERROR | 橙色 | 严重，功能受损 | 15分钟内 |
| P2 | WARNING | 黄色 | 警告，潜在问题 | 1小时内 |
| P3 | INFO | 蓝色 | 提示，需关注 | 工作时间 |

### 1.2 告警通道

| 通道 | 用途 | 配置 |
|------|------|------|
| Email | 正式通知、报告 | smtp配置 |
| Slack | 实时通知、协作 | webhook URL |
| SMS | 紧急告警 P0/P1 | 手机号列表 |
| PagerDuty | 值班通知 | integration key |
| Webhook | 自定义集成 | HTTP endpoint |

## 九、告警升级机制 (v1.2新增)

### 9.1 升级策略定义

告警升级机制确保高优先级问题得到及时响应，当首次响应未在规定时间内完成时自动升级。

```python
# 告警升级配置
ALERT_ESCALATION_CONFIG = {
    "enabled": True,
    "check_interval_seconds": 60,  # 每分钟检查一次
    
    # 升级链路: L1值班 -> L2技术负责人 -> L3架构师 -> L4值班经理
    "escalation_levels": [
        {"level": 1, "name": "L1_值班工程师", "timeout_minutes": 5},
        {"level": 2, "name": "L2_技术负责人", "timeout_minutes": 10},
        {"level": 3, "name": "L3_架构师", "timeout_minutes": 15},
        {"level": 4, "name": "L4_值班经理", "timeout_minutes": 30},
    ],
    
    # 自动升级触发条件
    "auto_escalate_conditions": [
        {"condition": "no_ack", "threshold_minutes": 5},
        {"condition": "no_resolution", "threshold_minutes": 30},
        {"condition": "same_issue_repeat", "count": 3, "window_hours": 1},
    ]
}
```

### 9.2 升级触发规则

| 触发条件 | 触发描述 | 升级动作 |
|----------|----------|----------|
| 首次响应超时 | P0告警发出后5分钟无确认 | → 升级到L2技术负责人 |
| 解决超时 | P0告警发出后30分钟未解决 | → 升级到L3架构师 |
| 重复告警 | 同一问题1小时内出现3次 | → 升级到L2技术负责人 |
| P1累计 | P1告警10分钟内累计≥3个 | → 升级到L2技术负责人 |
| 关联故障 | P0告警触发后2分钟 | → 同步通知L3架构师 |

### 9.3 升级状态追踪

```python
class EscalationTracker:
    """告警升级状态追踪器"""
    
    def __init__(self):
        self.active_escalations: Dict[str, EscalationState] = {}
        self.escalation_history: List[EscalationEvent] = []
    
    def start_escalation(self, alert_id: str, initial_level: int):
        """启动升级追踪"""
        self.active_escalations[alert_id] = EscalationState(
            alert_id=alert_id,
            current_level=initial_level,
            start_time=datetime.now(),
            ack_time=None,
            last_update_time=datetime.now(),
            escalation_count=0
        )
    
    def escalate(self, alert_id: str, reason: str) -> bool:
        """执行升级"""
        if alert_id not in self.active_escalations:
            return False
        
        state = self.active_escalations[alert_id]
        if state.current_level >= 4:  # 已达最高级别
            return False
        
        state.current_level += 1
        state.escalation_count += 1
        state.last_update_time = datetime.now()
        
        self.escalation_history.append(EscalationEvent(
            alert_id=alert_id,
            from_level=state.current_level - 1,
            to_level=state.current_level,
            reason=reason,
            timestamp=datetime.now()
        ))
        
        return True
    
    def should_auto_escalate(self, alert_id: str) -> Tuple[bool, str]:
        """检查是否应该自动升级"""
        if alert_id not in self.active_escalations:
            return False, ""
        
        state = self.active_escalations[alert_id]
        alert = self.get_alert(alert_id)
        
        # 检查响应超时
        if not state.ack_time:
            ack_elapsed = (datetime.now() - state.start_time).total_seconds() / 60
            timeout = ESCALATION_CONFIG["escalation_levels"][state.current_level]["timeout_minutes"]
            if ack_elapsed > timeout:
                return True, f"响应超时: 已过{ack_elapsed:.0f}分钟，超过SLA要求{timeout}分钟"
        
        # 检查解决超时
        resolution_timeout = SLA_CONFIG.get(alert.severity, {}).get("resolution_time", 60)
        total_elapsed = (datetime.now() - state.start_time).total_seconds() / 60
        if total_elapsed > resolution_timeout and alert.status != "resolved":
            return True, f"解决超时: 已过{total_elapsed:.0f}分钟，超过SLA要求{resolution_timeout}分钟"
        
        return False, ""
```

## 十、告警响应SLA详细规范 (v1.2新增)

### 10.1 SLA级别定义

| SLA等级 | 响应时间 | 解决时间 | 可用性目标 | 适用场景 |
|----------|----------|----------|------------|----------|
| **SLA-S (P0)** | 5分钟 | 30分钟 | 99.99% | 系统不可用、数据丢失风险 |
| **SLA-A (P1)** | 15分钟 | 2小时 | 99.9% | 核心功能受损、性能严重下降 |
| **SLA-B (P2)** | 1小时 | 8小时 | 99.5% | 非核心功能异常、潜在风险 |
| **SLA-C (P3)** | 工作时间 | 72小时 | 99.0% | 优化建议、观察项 |

### 10.2 各层级SLA详情

#### FLY-2 规则引擎 SLA

| 指标 | SLA目标 | 计算方式 | 告警阈值 |
|------|---------|----------|----------|
| 规则执行成功率 | ≥99.5% | 成功次数/总次数 | <99% 触发P1 |
| 规则执行延迟P99 | ≤500ms | 百分位统计 | >1000ms 触发P2 |
| 规则匹配延迟P99 | ≤200ms | 百分位统计 | >500ms 触发P2 |
| 缓存命中率 | ≥85% | 命中/总请求 | <70% 触发P2 |
| 死锁检测响应 | ≤1分钟 | 检测到处理完成 | >1分钟 触发P0 |

#### FLY-3 趋势分析 SLA

| 指标 | SLA目标 | 计算方式 | 告警阈值 |
|------|---------|----------|----------|
| 预测请求成功率 | ≥99% | 成功/总请求 | <97% 触发P1 |
| 预测准确度 | ≥75% | 预测准确率 | <70% 触发P2 |
| 卷积执行成功率 | ≥98% | 成功/总执行 | <95% 触发P1 |
| 预测延迟P99 | ≤3000ms | 百分位统计 | >5000ms 触发P2 |

#### FLY-5 工具中心 SLA

| 指标 | SLA目标 | 计算方式 | 告警阈值 |
|------|---------|----------|----------|
| 消息发布成功率 | ≥99.9% | 成功/总发布 | <99.5% 触发P1 |
| 消息消费延迟P99 | ≤500ms | 百分位统计 | >1000ms 触发P2 |
| RPC调用成功率 | ≥99.5% | 成功/总调用 | <99% 触发P1 |
| 资源分配成功率 | ≥99% | 成功/总分配 | <98% 触发P1 |
